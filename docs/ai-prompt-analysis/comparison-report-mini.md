## 5대 핵심 결론

### 1. 구조적 준수도: nano 불안정 → mini 완벽
nano에서 실패했던 세션 분리, 3컬럼 테이블, 카테고리 분배, 교차 참조가 **모두 해결**. 프롬프트 변경 없이 모델 교체만으로 구조적 품질이 대폭 향상.

### 2. JSON 출력 안정성: nano 불완전 → mini 완전
nano의 가장 치명적인 문제(뉴스카드 0개)가 해결. mini는 복잡한 JSON 구조(칼럼 2 + 카드 6 + 종목 3 + 키워드 10 + 감성)를 안정적으로 출력.

### 3. 길이는 여전히 미달 (67-89%)
프롬프트에서 "미달 시 실패"로 명시해도 mini는 목표 길이의 67-89%만 달성. 이는 모델의 간결함 선호 특성으로, 프롬프트 강화보다는 **분할 생성** 또는 **few-shot 예제** 제공이 더 효과적일 수 있음.

### 4. 교차 참조 품질 우수
mini는 "(관련 기사: 구체적 기사명)" 형태로 본문 곳곳에 교차 참조를 삽입. 이는 nano에서 전혀 불가능했던 기능이며, 프롬프트 강화의 효과가 mini에서만 나타남.

### 5. 비용 대비 효율 우수
nano 대비 약 40배 비용 증가($0.001 → $0.05/호출)이지만, 품질 향상은 비용을 충분히 정당화. 특히 뉴스카드 생성이 가능해진 것은 nano에서는 달성 불가능한 핵심 기능.
