# 뉴스데스크 기능 제안서

> **작성 목적**: 팀 논의를 위한 아이디어 공유  
> **작성일**: 2026년 2월 8일

## 📌 핵심 아이디어

기존의 정적인 뉴스 페이지가 아니라, **AI 에이전트가 매일 뉴스를 분석하고 상황에 맞는 레이아웃과 컴포넌트를 스스로 결정해서 생성**하는 동적 대시보드를 만들어보면 어떨까요?

---

## 🎯 제안 배경

### 현재 일반적인 뉴스 페이지
```
[고정된 레이아웃]
📰 뉴스 카드 1
📰 뉴스 카드 2
📰 뉴스 카드 3
```
→ 매일 같은 형식, 지루함

### 제안하는 방식
```
[AI가 판단해서 구성]

🔴 급등락 날:
  → 긴급 분석 칼럼 + 호재/악재 차트 + 섹터별 히트맵

🟢 평온한 날:
  → 카드뉴스 3개 + 키워드 워드클라우드 + 관심종목 순위

🔵 특정 이슈 집중:
  → 심층 분석 글 + 타임라인 + 관련 종목 영향도
```
→ **상황에 맞는 동적 구성**

## 🏗️ 대략적인 구조 (예시)

### 페이지 레이아웃 (안)

```
┌─────────────────────────────────────────────────┐
│  📊 상단: 증시 차트 (고정)                       │
│  코스피/나스닥/S&P500/우리펀드 - 토글 가능      │
└─────────────────────────────────────────────────┘

┌──────────────────────┬──────────────────────────┐
│                      │                          │
│  왼쪽 (메인 콘텐츠)  │   오른쪽 (시각화)        │
│                      │                          │
│  [AI가 선택]         │   [AI가 선택]            │
│  - 카드뉴스 or       │   - 워드클라우드 or      │
│  - 칼럼 or           │   - 호재/악재 차트 or    │
│  - 불릿 요약 or      │   - 순위 바차트 or       │
│  - 데이터 테이블     │   - 타임라인 등          │
│                      │                          │
└──────────────────────┴──────────────────────────┘
```

### 핵심 포인트
- **상단 차트 영역**: 고정 (사용자가 직접 토글 조작)
- **하단 콘텐츠 영역**: 유동적 (AI가 매일 다르게 구성)

## 🤖 AI 에이전트의 역할 (개념)

```
1. 크롤링 (05:30)
   ↓
2. 뉴스 분석
   - 키워드 추출
   - 감성 분석 (호재/악재/중립)
   - 주요 이슈 파악
   ↓
3. 레이아웃 결정 (핵심!)
   "오늘은 특정 이슈가 집중되니까 칼럼 형식이 좋겠다"
   "키워드가 다양하니 워드클라우드를 보여주자"
   "호재 뉴스가 많으니 감성 차트를 넣자"
   ↓
4. 컴포넌트 생성
   선택한 도구로 실제 콘텐츠 생성
   ↓
5. DB 저장
   ↓
6. 프론트엔드는 그냥 렌더링
```

## 🛠️ AI가 사용할 수 있는 도구 (예시)

이건 확정이 아니라 **가능성을 보여주는 예시**입니다. 실제로는 논의 후 조정 가능:

### 텍스트 컴포넌트
- **카드뉴스**: 3-5개 핵심 뉴스 카드
- **칼럼**: 긴 분석 글 (심층 분석 필요할 때)
- **불릿 요약**: 빠른 스캔용
- **타임라인**: 시간 순서가 중요한 이벤트

### 차트 컴포넌트
- **워드클라우드**: 많이 나온 키워드
- **호재/악재 차트**: 파이/바 차트로 시장 분위기
- **순위 차트**: 관심 종목, 상승/하락률 TOP N
- **히트맵**: 섹터별 등락
- **데이터 테이블**: 주요 경제 지표

## 💡 왜 이게 좋을까? (개인 생각)

### 1. 사용자 경험
- 매일 다른 형식 → 지루하지 않음
- 상황에 맞는 정보 → 더 유용함
- "오늘은 급등락이니 긴급 분석이 먼저다" → 맥락 있는 정보

### 2. 차별화
- 일반 뉴스 사이트: 다 비슷비슷
- 우리: AI가 알아서 구성하는 독특한 경험

### 3. 확장 가능성
- 새로운 차트 타입 추가 → AI가 알아서 사용
- 새로운 데이터 소스 추가 → 자동으로 반영
- 수동으로 레이아웃 짜지 않아도 됨

## 📅 구현 단계 (참고용, 조정 가능)

### Phase 1: 기본 뉴스 크롤링 + 고정 레이아웃
- 네이버 금융 크롤링
- 고정된 카드뉴스 3개
- 차트 영역 (토글 기능)
- **목표**: 기본 뉴스데스크 작동

### Phase 2: AI 분석 추가
- AI로 뉴스 요약
- 키워드 추출
- 감성 분석
- **목표**: AI 파이프라인 구축

### Phase 3: 동적 레이아웃 (핵심!)
- AI가 레이아웃 결정
- 2-3개 컴포넌트 타입
- 동적 렌더링
- **목표**: 레이아웃 자동 구성

### Phase 4: 컴포넌트 확장
- 더 많은 차트 타입
- 더 많은 텍스트 타입
- 레이아웃 조합 다양화
- **목표**: 풍부한 표현력

## 🚧 잠재적 이슈 (미리 생각해볼 것들)

### 1. AI가 이상한 선택을 하면?
- 폴백 레이아웃 필요
- AI 결정에 rule-based 검증 추가?
- 수동 오버라이드 기능?

### 2. 데이터가 부족하면?
- 뉴스가 너무 적은 날
- 크롤링 실패
- 최소 컴포넌트 보장?

### 3. 사용자 혼란
- 매일 다른 레이아웃 → 익숙해지기 어려움?
- 일부는 고정, 일부만 동적?
- 사용자 설정으로 고정 레이아웃 선택?

### 4. 성능 문제
- AI 응답 느리면 새벽 6시 전에 안 끝남
- Batch API 사용? (50% 할인이지만 느림)
- 캐싱 전략?
